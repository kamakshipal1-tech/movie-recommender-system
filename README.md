# 🎬 Movie Recommender System

## Overview

This project is an end-to-end **Content-Based Movie Recommendation System** that suggests movies similar to a user's selected movie based on their content rather than user ratings or viewing history. The application analyzes movie metadata such as genres, keywords, cast, crew, and overview to identify similar movies using Natural Language Processing (NLP) techniques and Cosine Similarity.

The final model has been deployed as an interactive web application using **Streamlit**, where users can select any movie from a dropdown menu and instantly receive five similar movie recommendations along with their official posters fetched dynamically using the **TMDB API**.

---

## Business Problem

With thousands of movies available across streaming platforms, discovering relevant content has become increasingly challenging. Users often spend a significant amount of time searching for movies instead of watching them.

Recommendation systems help solve this problem by providing personalized suggestions, thereby improving:

* User engagement
* Content discovery
* Customer satisfaction
* User retention
* Overall watch time

This project demonstrates how machine learning can be used to build an intelligent recommendation engine that recommends movies based solely on their content.

---

## Recommendation Approach

Recommendation systems are broadly classified into three categories:

### Content-Based Filtering

Recommendations are generated by comparing the characteristics of movies. Movies having similar genres, cast members, directors, keywords and storylines are considered similar.

**This project uses Content-Based Filtering.**

### Collaborative Filtering

Recommendations are generated based on user behaviour such as ratings, watch history and preferences.

### Hybrid Recommendation System

A combination of both content-based and collaborative filtering approaches. Most commercial streaming platforms use hybrid recommendation systems.

## Dataset

The model is built using the **TMDB 5000 Movie Dataset**, consisting of two datasets:

* Movie metadata
* Movie credits

Both datasets are merged to obtain complete information about each movie.

---

## Data Preprocessing

The following preprocessing steps were performed before building the recommendation model:

* Merged movie and credits datasets
* Removed duplicate records
* Selected relevant features
* Handled missing values
* Extracted director information
* Extracted top cast members
* Processed genres and keywords
* Removed unnecessary spaces
* Converted all textual information into a single feature called **tags**

---

## Feature Engineering

Instead of comparing multiple columns individually, important information from different columns was combined into a single textual feature called **tags**.

The tags contain information such as:

* Genres
* Keywords
* Overview
* Top cast
* Director

This creates a meaningful textual representation of every movie that can be used for similarity comparison.

---

## Text Vectorization

Machine learning algorithms cannot process raw text directly. Therefore, the textual tags were converted into numerical vectors using **CountVectorizer**.

### Bag of Words

The Bag of Words technique converts text into vectors based on the frequency of words appearing in each movie's tags.

Instead of understanding language semantically, it represents every movie as a high-dimensional numerical vector, enabling mathematical comparison between movies.

---

## Similarity Measure

Once every movie is converted into a vector, similarity between movies is calculated using **Cosine Similarity**.

Cosine Similarity measures the angle between two vectors rather than the physical distance between them.

This approach is preferred over Euclidean Distance because text documents often differ in length. Cosine Similarity focuses on the direction of vectors instead of their magnitude, making it more suitable for text-based recommendation systems.

For every selected movie, similarity scores are calculated against all other movies, and the five movies with the highest similarity scores are recommended.

---

## Model Workflow

The complete workflow followed in this project is:

1. Import the datasets
2. Perform data preprocessing
3. Create the tags feature
4. Convert text into numerical vectors using CountVectorizer
5. Compute cosine similarity between all movie vectors
6. Store the processed movie list and similarity matrix using Pickle
7. Build an interactive Streamlit application
8. Fetch movie posters dynamically using the TMDB API
9. Deploy the application for public use

## Deployment

The recommendation system has been deployed using **Streamlit Community Cloud**.
